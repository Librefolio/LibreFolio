# F-047 â€” Transaction List Page

## What it does
The `/transactions` page showing all transactions the current user has access to across all their brokers. Built on the generic `DataTable` component with column header filters (multi-enum, currency-stack range sliders, date pickers, text search). Supports bulk delete, TransferPromote, and Staging Modal launch. TRANSFER/FX_CONVERSION pairs always rendered adjacent (giver above / receiver below) via the always-pair-adjacent algorithm (see [[concepts/always-pair-adjacent]]).

## Architecture notes

### Client-side filtering
All filtering is 100% client-side. `GET /transactions` loads ALL user transactions (no filter params). Explicit **Refresh** button (â†ş) triggers reload. See [[decisions/transactions-client-side-filtering]].

### Always-pair-adjacent rendering
TRANSFER/FX_CONVERSION pairs always adjacent: giver above, receiver below. Secondary fetch for ghost rows when partner absent from main result. Conditional: only when no column filter and no sort active (with filter/sort â†’ `isGrouped=false`, independent rows). In flat mode, adjacency preserved via `sortIndex` algorithm (giver gets N, receiver gets N+0.5). See [[concepts/always-pair-adjacent]].

### Entity store (asset + broker)
`assetStore` and `brokerStore` use `createEntityStore<T>()` factory for shared cache with proper invalidation. See [[concepts/entity-store-pattern]].

### Modals
- `TransactionFormModal.svelte` â€” single-row CRUD (create/edit/duplicate/view) + **dual-form mode** for paired transactions (FX, Transfer Asset, Transfer Cash). See [[decisions/dual-transaction-form-design]]
- `TransactionBulkModal.svelte` â€” **Mode-less** unified batch editor: create + edit + delete in one modal. Bulk delete opens with `initialStatus: 'delete'`. See [[features/F-048]], [[decisions/bulkmodal-mode-removal]]
- `TransactionDeleteModal.svelte` â€” Rich single/paired delete confirmation with 3 layouts (A/B/C), validate-now, error banners
- `PromotePairWizardModal.svelte` â€” 3-step wizard for pair promotion (replaces legacy `TransferPromoteModal`)

### New filter variants (DataTable)
- `multi-enum` â€” checkbox list with search (transaction type, tags)
- `currency-stack` â€” per-currency range sliders with flag+symbol+code title (`getCurrencyMinMaxByCode(column)`)
- `DataTable.fullData` prop â€” stable min/max boundaries (don't shrink as filter narrows)

### Tooltip rule
All tooltips inside DataTable cells use `<Tooltip.svelte>` via CustomCell. `title=""` HTML attributes prohibited. See [[decisions/datatable-tooltip-custom-cell]].

### Pulse animation (GoTo linked pair)
Click `đź”—` icon â†’ DOM-direct `classList.add/remove` + `offsetWidth` reflow to restart CSS animation on `<td>` (`tr.tx-row-highlight > td`). Capture-phase event delegation for `đź”—`/`â—Źevt` click events.

### Context menu (Round 6)
Right-click (desktop) / long-press (mobile) on any row opens `ContextMenu.svelte` with same actions as the row action column (edit, clone, delete, view, split). Enabled by default on ALL DataTables via `enableContextMenu=true` prop. See [[decisions/context-menu-all-tables]].

### Broker access gating (Round 6)
Row actions (edit, delete, clone) gated by `getPairedAccessLevel()` â€” paired transactions use min(role_A, role_B). VIEWER/no-access â†’ view only. `brokerRoleHelpers.ts` provides `getRoleIcon()`, `canEditWithRole()`. `BrokerBadge` shows role icons (Crown/Pencil/Eye/Lock). See [[decisions/broker-access-min-paired]].

### URL filter sync (Round 6)
All filter state bidirectionally synced to URL params (id, qty, broker multi, asset multi, description). `navigationStore` tracks full URL (pathname+search) â€” back button preserves filters. Race condition fix: skip init emissions to avoid clearing user-set filters.

### Asset clickable (Round 6)
Asset names are SPA links (`goto(/assets/{id})`) with `â†—` icon on hover. Uses `data-asset-navigate` attribute and capture-phase click delegation.

### Description column (Round 6)
New `description` column: `type: 'text'`, filterable, sortable, URL-synced. Truncation with `...` and tooltip via `HtmlCell.tooltip` for texts > 20 chars.

### Soft reload (Round 6)
`reload({soft: true})` preserves DataTable filter/sort state after save operations from BulkModal/FormModal/Delete. Hard reload via â†ş button resets filters.
