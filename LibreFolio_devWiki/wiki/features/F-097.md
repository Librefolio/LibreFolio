# F-097 — WAC (Weighted Average Cost)

## What it does
Calculates the weighted average cost basis for assets using all historical BUY prices and incoming TRANSFER cost_basis values, with automatic FX conversion for multi-currency portfolios. Auto-fills `cost_basis_override` on TRANSFER receiver transactions when no explicit value is provided. Provides inline preview in validate response and auto-applies in commit.

## Layer breakdown
**Backend**:
- `compute_weighted_avg_cost(session, broker_id, asset_id, as_of_date, asset_currency)` → `WACResult`
- FX cross-currency: converts each BUY/TRANSFER amount via FX rate at TX date
- Target currency determination: **last acquisition's currency** (deterministic, [[decisions/wac-target-currency-last-acquisition]])
- Override via sentinella: `cost_basis_override: {code: "KRW", amount: "0"}` = "calculate WAC in KRW"
- Auto-calc at commit: applied to TRANSFER receiver (qty>0) when `cost_basis_mode: auto`
- `cost_basis_override` is Currency object `{code, amount}` ([[decisions/cost-basis-currency-object]])
- `WACResult` schema: `wac: Optional[Currency]`, `qualifying_txs: list[WACQualifyingTX]`, `missing_pairs: list[WACMissingPairInfo]`
- `WACQualifyingTX` includes `fx_info` (FxBackwardFillInfo), `original_unit_cost`, `original_currency`, `fx_rate_used`
- `WACMissingPairInfo`: `{pair: str, dates: list[date]}` — enriched with specific missing dates
- Validation issue `WAC_FX_UNAVAILABLE` emitted when auto mode fails due to missing FX
- `COST_BASIS_REQUIRED` validation (step 6d) blocks TRANSFER/ADJUSTMENT(qty>0) without cost_basis

**Frontend**:
- `WacPreviewSection.svelte` — controlled component (mode from parent, no internal state)
- Auto/Manual toggle; chip valuta selector for currency override (stays in auto)
- Qualifying TX table with FX conversion arrows (original → converted), stale warning (⚠️), tooltip
- AssetEventPicker: `AssetEventPicker.svelte` — links TX to declared asset events (DIVIDEND, INTEREST)
- FxSyncModal integration: parent owns modal, child calls `onOpenFxSync` ([[decisions/fxsyncmodal-parent-ownership]])
- Blur detection: `formatDecimalForDisplay()` comparison ([[decisions/blur-detection-format-string-comparison]])
- `useValidateScheduler`: manual triggers bypass antiBounce 10s (for post-sync re-validate)
- Validation banner: inline sync button for `wacFxUnavailable` issues
- FormModal props unified: single `items: FormModalItems | null` prop

## Implementation notes
- **Breaking change**: old `"42.50"` string format for cost_basis → 422 (no backward compatibility)
- **Fail-safe**: if even one FX conversion is impossible → `WACResult(wac=None)`, missing_pairs populated with dates
- **Zero-quantity edge case**: `total_qty == 0` with no FX errors → `Currency(code=target, amount="0")`
- **WAC inline in validate/commit** ([[decisions/wac-inline-validate-commit]]): no standalone endpoint in editing flow
- **cost_basis_mode**: explicit `'auto' | 'manual'` field in payload — prevents feedback loop
- **Target currency = last acquisition** (not majority): deterministic, no tie-breaking needed
- **Sentinella override**: `{code: "X", amount: "0"}` + mode auto = "calculate in currency X"
- **FX missing → NOT force manual**: informative error, user can sync FX and stay in auto
- **Anti-bounce 10s**: exists in useValidateScheduler to prevent validate spam, but manual trigger bypasses it
- **PromoteMergeModal WAC section**: was dead code (no caller passed WAC props) — removed in WacFxEnrich
- **Issue index mapping**: `buildOpsIndexMap()` translates `operation:index` → tempId for correct row highlighting
- **Na/Nb notation**: paired rows show "Riga 5a"/"Riga 5b" in issue banner

## Dead ends / Ideas explored
- Initially considered keeping backward compat with bare decimal string — rejected because Currency object provides unambiguous currency association
- Considered persisting `wac_info` to DB — rejected because it's a derived value
- **Standalone `/wac-preview` endpoint**: caused feedback loops. Replaced with WAC inline in validate/commit
- **`null` as auto sentinel**: ambiguous. Replaced with explicit `cost_basis_mode` field
- **ReactiveWacBulkModal v1-v3**: failed due to reactivity issues. Rearchitected via v4-v6
- **WacPreviewSection with internal state**: caused mode loss on re-open. Replaced with fully controlled component
- **Majority currency for target**: non-deterministic on tie, confusing with single TX flip
- **forcedManual $effect on FX missing**: broke user flow (couldn't sync FX without switching to manual)
- **FxSyncModal inside WacPreviewSection**: z-index issues, duplicated instances
- **Navigate to /fx page for missing pair**: loses form context entirely (mobile incompatible)
